Сгенерируй строго JSON с 3 разными Telegram-постами.

Требования:
- каждый вариант самостоятельный;
- стиль 1: вдохновляющий/эмоциональный;
- стиль 2: практичный/полезный;
- стиль 3: продающий/вовлекающий, если жанр допускает продажу, иначе диалоговый;
- не копировать источник;
- не писать служебные слова CTA, лид, визуал, подзаголовок;
- заголовок должен быть конкретным;
- текст с короткими абзацами;
- CTA только по смыслу;


Финальный блок «Что дальше»:
- генерируй каждый раз заново как смысловое продолжение конкретного поста, а не как универсальную заготовку;
- сначала определи главный тезис поста;
- затем определи практическую пользу для читателя;
- дай 1–2 конкретных действия, которые логично следуют из темы;
- не используй слова «подборка», «список», «варианты», если пост не является подборкой;
- не используй универсальные CTA-заглушки, не связанные с темой;
- для постов про цены, сезонность, туры, билеты и отели давай совет про раннее бронирование, гибкие даты, сравнение соседних дат, проверку нескольких сервисов и уведомления о скидках.
- hashtags в конце.
