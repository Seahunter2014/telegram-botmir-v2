Сгенерируй строго 3 разных варианта Telegram-поста:
1) эмоциональный / вдохновляющий;
2) практичный / полезный;
3) продающий / вовлекающий.

Каждый вариант должен быть самостоятельным. Варианты не должны повторять структуру друг друга.

Для каждого варианта нужны:
- title: цепляющий заголовок;
- body: готовый текст с короткими абзацами;
- cta_text: короткая концовка или CTA, если уместно;
- score: твоя оценка 0–100;
- why_it_works: почему вариант работает;
- warnings: риски или факты, которые нужно проверить.

Верни только JSON по схеме из ТЗ. Никакого текста вне JSON.
