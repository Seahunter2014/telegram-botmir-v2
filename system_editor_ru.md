# Чек-лист работы финального бота

## Проверка после деплоя

1. `/version` должен показать: `31.05.26 FINAL NEWSROOM`.
2. В версии должны быть видны: 28 тем, 14 источников, 9 сервисов.
3. `/topics` должен вывести полный список тем.
4. `/sources` должен вывести источники.
5. `/services` должен вывести партнёрские сервисы.
6. `/test` должен собрать 3 разных варианта поста.
7. В тексте не должно быть служебных слов: `сигнал`, `жанр`, `destination_post`, `weekend_trip`.
8. В тексте не должно быть старых шаблонов.
9. Текст не должен обрезаться: фото отправляется отдельно, текст отдельно.
10. Кнопки должны соответствовать теме.

## Как должен работать `/test`

- бот собирает темы из источников;
- выбирает лучший сигнал;
- определяет тему;
- создаёт 3 варианта;
- показывает оценку качества;
- даёт кнопки: отправить в тест или опубликовать.

## Как должен работать `/publish`

- бот собирает новую тему;
- выбирает лучший вариант по QA-score;
- отправляет фото;
- отправляет текст с кнопками;
- записывает публикацию в `data/state.json`.

## Что считается ошибкой

- появилась абракадабра вида `битая кириллица`;
- появилась фраза `Есть направления, которые`;
- появился служебный жанр в публичном тексте;
- нет кнопок там, где они должны быть;
- бот выдумывает цену, дату или отель;
- один и тот же источник идёт подряд слишком часто.
