# 1. Главная цель проекта

Нужно разработать не рерайтер, не копипастер, не шаблонный генератор и не агрегатор ссылок.

Нужно разработать **Telegram-бота AI-редакцию**, который помогает travel-каналу «Мир на ладони» расти естественно:

- за счёт интересных постов;
- за счёт сохранений;
- за счёт пересылок друзьям;
- за счёт реакций и комментариев;
- за счёт регулярности публикаций;
- за счёт чередования форматов;
- за счёт полезности для русскоязычных путешественников;
- за счёт аккуратных переходов по партнёрским ссылкам;
- за счёт контента, который Telegram-алгоритмы могут воспринимать как живой, полезный и вовлекающий.

Источник — это сырьё.  
Пост — это авторский редакционный продукт.

Каждый пост должен отвечать на вопрос:

> Почему читателю стоит это прочитать, сохранить, переслать, обсудить или открыть по ссылке?

Если ответа нет — тема не публикуется в исходном виде, а бот ищет другой угол, другую тему или fallback-тему.

---

# 2. Главный принцип работы

Бот не обслуживает источник.  
Бот обслуживает интерес канала и его аудитории.

Источник может дать:

- новость;
- цену;
- маршрут;
- страну;
- город;
- событие;
- визу;
- идею;
- подборку;
- фото-повод;
- практический сигнал.

Но итоговый пост должен быть написан заново:

- живым русским языком;
- с Telegram-ритмом;
- с сильным заголовком;
- с понятным первым экраном;
- с конкретикой;
- с уместными эмодзи;
- с логичной концовкой;
- с CTA только по смыслу;
- с картинкой или видео, которые усиливают тему;
- без копирования источника;
- без школьного пересказа;
- без шаблонной «нейросетевой воды».

# 3. Обязательное правило результата

Если нет глобального технического сбоя Telegram, OpenAI, сервера или критических переменных окружения, бот **всегда должен найти тему, сгенерировать пост и попытаться его опубликовать**.

Логика обязательного результата:

```text
основные источники
→ резервные источники
→ evergreen-источники
→ fallback-темы
→ уникальный угол раскрытия
→ генерация 3 вариантов
→ выбор лучшего
→ публикация
→ отчёт админу
```

Бот не должен молчать. Если публикация не состоялась, бот обязан объяснить админу причину.

# 5. Что бот должен делать

Бот должен:

1. Обходить источники по кругу.
2. Не залипать на одном источнике.
3. Получать свежие сигналы.
4. Проверять доступность каждого источника.
5. Сохранять source health: работает / не работает / сколько сигналов получено / ошибка.
6. Отбрасывать мусорные темы.
7. Отбрасывать не travel-темы.
8. Отбрасывать чужие рекламные анонсы без travel-ценности.
9. Проверять свежесть.
10. Проверять дубли по URL, заголовку, смыслу, городу, стране, жанру и структуре.
11. Классифицировать жанр.
12. Оценивать тему по score.
13. Выбирать лучшую тему под текущий слот.
14. Создавать редакционный brief.
15. Генерировать 3 разных варианта поста.
16. Проверять каждый вариант.
17. Убирать шаблонность и AI-smell.
18. Подбирать CTA.
19. Подбирать реферальные ссылки.
20. Если реферальные ссылки неуместны — ставить универсальные вовлекающие кнопки.
21. Подбирать медиа на русском и английском языке.
22. Если фото не найдено — генерировать тематическую картинку или использовать универсальный travel-визуал.
23. В ручном режиме показывать 3 варианта.
24. В автопостинге выбирать лучший вариант сам.
25. Публиковать в один или несколько каналов.
26. Сохранять историю публикаций.
27. Показывать понятный статус.
28. Показывать причину, если пост не опубликован.
29. Уведомлять администратора об ошибках автопостинга.
30. Использовать fallback-темы, если источники недоступны.
31. Никогда не повторять один и тот же смысл.
32. Учиться на истории публикаций и реакциях, когда analytics будет подключена.

# 6. Что бот не должен делать

Бот не должен:

- копировать источник;
- пересказывать источник;
- писать пост ради источника;
- рекламировать чужой Telegram-канал;
- вести кнопку на чужой Telegram-пост, если это не осознанный source-credit в админском режиме;
- публиковать чужие эфиры, вебинары, вакансии, IT-карьеру, крипту, airdrop;
- писать сухой пресс-релиз;
- писать школьный пересказ;
- вставлять ссылки ради галочки;
- вставлять билеты там, где тема не про поездку;
- вставлять отели там, где тема про визу, карты или документы;
- публиковать одинаковые структуры подряд;
- публиковать одинаковые темы подряд;
- писать технические слова вроде `weekend_trip`, `flight_deal`, `signal`, `CTA`, `лид`, `визуал` в публичном посте;
- использовать старые шаблонные фразы;
- публиковать битую кодировку;
- публиковать случайную картинку;
- публиковать «белую карточку» вместо нормального фото;
- молчать, если автопостинг не сработал;
- публиковать Markdown-разметку, которая не отрендерилась в Telegram.

# 8. Fallback-темы, если источники не работают

Если ни один источник не работает или все темы забракованы, бот обязан не молчать, а сгенерировать пост на одну из широких evergreen-тем.

Важно: fallback — это не повтор старого поста.  
Одна широкая тема может раскрываться десятками способов.

Например тема «курорты Средиземноморья» может раскрыться как:

- топ-10 курортов Средиземноморья;
- курорты Турции на Средиземном море;
- отдельный пост про Анталью;
- сравнение Кипра, Греции и Турции;
- бюджетные пляжи Средиземноморья;
- семейные курорты Средиземноморья;
- куда ехать осенью;
- куда ехать без машины;
- где лучше море;
- где дешевле отели.

Бот должен хранить не только название fallback-темы, но и угол раскрытия, чтобы не повторяться.

# 11. Правила выбора тем

Бот выбирает темы по критериям:

1. Актуальность для россиян:
   - визы;
   - маршруты из РФ;
   - карты и оплата;
   - сезонность отпусков;
   - доступность направления.

2. Универсальность:
   - тема интересна широкой аудитории;
   - фокус на зарубежные направления, но Россия допустима, если тема сильная.

3. Визуальная привлекательность:
   - тему можно хорошо иллюстрировать;
   - город, море, горы, архитектура, еда, маршрут.

4. Практическая польза:
   - лайфхак;
   - подборка;
   - чек-лист;
   - маршрут;
   - сравнение;
   - цена;
   - сезон.

5. Позитивный тон:
   - без нагнетания;
   - вместо «опасные районы» — «как сделать поездку комфортнее».

6. Без политики:
   - не обсуждать конфликты;
   - не спорить о территориях;
   - не писать политические выводы;
   - ограничения и визы описывать нейтрально и практично.

7. Разнообразие форматов:
   - топ-списки;
   - гайды;
   - истории;
   - мини-исследования;
   - офферы;
   - события;
   - полезные карточки;
   - маршруты.

8. Вовлекающий потенциал:
   - вопрос;
   - выбор;
   - реакция;
   - комментарии;
   - сохранить;
   - переслать другу.

9. Экспертность:
   - не поверхностный пересказ;
   - конкретные детали;
   - полезный вывод.

10. Алгоритмический потенциал Telegram:
    - яркий первый экран;
    - цепляющий заголовок;
    - не слишком длинный пост;
    - понятная структура;
    - тема, которую хочется переслать;
    - качественная картинка;
    - регулярность.

# 12. Правила оформления Telegram-постов

Telegram не даёт менять размер шрифта, поэтому визуальная иерархия создаётся через:

- жирный заголовок;
- короткий лид;
- пустые строки;
- разделители;
- подзаголовки;
- списки;
- умеренные эмодзи;
- кнопки;
- хештеги.

## 12.1. Базовая структура

```text
🌎 **Заголовок поста**

*Интригующий лид на 2–3 предложения. Он должен объяснить, почему пост стоит читать дальше.*

──────✦──────

📌 **Блок 1**
Короткий абзац по теме.

🏖️ Пункт 1: описание + лайфхак.
✈️ Пункт 2: как добраться.
💰 Пункт 3: примерные цены.
🗓️ Пункт 4: лучший сезон.

──────✦──────

💡 **Лайфхаки и советы**
✅ Совет 1.
✅ Совет 2.
⚠️ Важно: осторожная формулировка.

──────✦──────

💬 **CTA**
Сохраните, отправьте другу, напишите в комментариях или проверьте предложение — только если это уместно.

#мирналадони #турджин #Туры #путешествиязаграницу #travel
```

Эта структура не должна использоваться механически в каждом посте. Бот выбирает блоки по жанру.

## 12.2. Заголовок

Заголовок:

- всегда отдельной первой строкой;
- выделяется жирным;
- может начинаться с 1 эмодзи;
- должен быть конкретным;
- не должен быть длиннее 1–2 строк в Telegram;
- должен содержать место, цифру, цену, выгоду, интригу, маршрут, сезон или проблему.

Формат:

```text
🌴 **10 мест в Марокко, где чувствуешь себя как в сказке**
```

## 12.3. Лид

Лид — 2–3 предложения после заголовка.

Лид может быть обычным текстом или курсивом.

Функция лида:

- зацепить внимание;
- объяснить пользу;
- дать обещание конкретики;
- не повторять заголовок;
- не начинаться шаблонно.

## 12.4. Разделители

Основной разделитель:

```text
──────✦──────
```

Использовать:

- между крупными смысловыми блоками;
- не чаще 3–5 раз в одном посте;
- не использовать после каждого абзаца.

Короткие посты до 800 символов могут быть без разделителей.

## 12.5. Подзаголовки

Подзаголовки выделяются жирным и могут начинаться с эмодзи:

Для подборок:

Для советов:

## 12.7. Эмодзи

Норма:

- 1 эмодзи в заголовке;
- 1 эмодзи в подзаголовке;
- эмодзи-маркеры в списках;
- всего обычно 3–8 эмодзи на пост.

Нельзя:

- ставить эмодзи в каждой строке без смысла;
- заменять эмодзи смыслом;
- делать пост похожим на рекламную листовку.

## 12.10. Медиа-блок

В опубликованном Telegram-посте не нужно писать служебный блок:

```text

видео: ...
```

Такой блок можно показывать только в админском preview.

В реальном посте бот должен:

- прикрепить фото/видео;
- либо использовать сгенерированную travel-картинку;
- либо использовать универсальный travel-визуал;
- не писать пользователю техническое описание картинки.

## 12.11. Длина поста и разбиение на части

Если текст с фото не проходит по длине caption Telegram, бот **не публикует фото отдельно**.

Правильное поведение:

1. Бот прикрепляет фото к первой части поста.
2. В первой части публикует начало текста.
3. В конце первой части пишет:

```text
Продолжение следует 👇
```

4. Следующим сообщением публикует вторую часть поста.
5. Вторая часть должна начинаться логично, без повторения заголовка, например:

```text
**Продолжение**
```

6. Хештеги ставятся в конце второй части.
7. CTA и кнопки ставятся там, где они логичнее: чаще во второй части, если это итоговый призыв.
8. Если пост делится на 2 части, обе части должны быть самостоятельными по читаемости и не обрываться посреди предложения.

Запрещено:

- публиковать фото отдельным пустым сообщением;
- публиковать обрезанный caption;
- разбивать текст посреди слова или предложения;
- повторять одинаковый CTA в обеих частях;
- терять кнопки при разбиении.

## 12.12. Форматы по длине

Короткий пост:

- 600–1200 символов;
- новость, оффер, быстрый совет.

Средний пост:

- 1200–2500 символов;
- основной формат канала.

Длинный пост:

- 2500–4000 символов;
- гайды, маршруты, топ-списки;
- при публикации с фото делится на 2 части по правилам выше.

# 13. Медиа: фото и видео

Медиа должно усиливать пост.

## 13.1. Правила медиа

1. Если пост про конкретный город — картинка должна быть узнаваемой.
2. Если пост про курорт — море/курорт должны соответствовать теме.
3. Если пост про еду — картинка должна быть аппетитной и релевантной.
4. Если пост про гайд — можно использовать нейтральное travel-фото или карточку.
5. Если нормального медиа нет — лучше сгенерировать картинку или использовать универсальный travel-визуал, чем публиковать мусор.
6. Чужие Telegram-картинки не использовать автоматически.
7. Проверять лицензию источника.
8. Хранить `media_source`, `media_author`, `license_url`, если доступно.

## 13.2. Поиск медиа

Бот ищет фото/видео по цепочке:

```text
RU query
→ EN query
→ broader EN query
→ location-based query
→ generated image
→ neutral travel fallback
```

Пример:

```text
Марокко пляжи Эс-Сувейра
→ Morocco Essaouira beach
→ Morocco ocean old town travel
→ Morocco travel landscape
→ generated image about Morocco travel
→ universal premium travel image
```

## 13.3. Бесплатные источники фото

1. Pexels.
2. Unsplash.
3. Pixabay.
4. ISO Republic.
5. StockSnap.io.

## 13.4. Бесплатные источники видео

1. Pexels Video.
2. Pixabay Video.
3. Mixkit.
4. Videezy — только если конкретный файл разрешён для нужного использования.
5. Pond5 Free — только free-коллекция и после проверки лицензии.

## 13.5. Генерация изображения

Если релевантное фото не найдено, бот генерирует красивую картинку:

- на тему поста;
- либо на близкую travel-тему;
- без текста, если текст может исказиться;
- в стилистике современного travel-медиа;
- без логотипов чужих брендов;
- без имитации реальных людей без разрешения;
- без политических, спорных или чувствительных элементов.

# 15. Ротация

Бот не должен публиковать подряд:

- один источник;

- тот же город — не раньше чем через 5 публикаций;
- тот же fallback base_topic — не раньше чем через 14 дней;

# 16. Scoring

Тема получает 0–100.

Параметры:

- свежесть: 0–20;
- конкретика: 0–20;
- актуальность для россиян: 0–15;

- практическая польза: 0–10;
- вовлекающий потенциал: 0–10;
- реферальная пригодность: 0–10;

1. расширить источники;
2. ослабить слот;

# 18. Диагностика

Бот не должен молчать.

После `/run_once` бот обязан прислать админу отчёт:

```text
Запуск начат
Источников проверено: N
Работают: N
Ошибки источников: N
Сигналов найдено: N
После фильтра travel-ценности: N
После дедупликации: N
Кандидатов для GPT: N
GPT: успешно / ошибка
Медиа: найдено / сгенерировано / fallback / ошибка
Публикация: успешно / ошибка / задержка Telegram
Каналы: список
Итог: опубликовано / пропущено / ошибка
```

После автопостинга по расписанию:

- если опубликовано — можно ничего не писать админу или отправить краткий отчёт;
- если не опубликовано — обязательно отправить админу причину;
- если Telegram задержал публикацию — бот должен отразить это в статусе как retry/delay, а не как молчаливый сбой.

В `/status` показывать:

- автопостинг;
- расписание;
- таймзону;
- следующий запуск;
- последний запуск;
- последний результат;
- последние ошибки источников;
- последний опубликованный пост;
- каналы;
- тестовый канал.

# 19. Надёжность и защита от зависаний

Бот должен:

- иметь таймаут на каждый источник;
- иметь общий таймаут на сбор источников;
- не блокировать event loop Telegram-бота;
- использовать retry для Telegram publish;
- если фото не отправилось — повторить без фото или с fallback-изображением;
- если кнопка битая — убрать кнопку и опубликовать пост без неё;
- если один канал не принял пост — попробовать остальные;
- записать результат по каждому каналу;
- не терять state при перезапуске Railway;
- не запускать две автопубликации одновременно.

13. Вдохновляющий пост может быть без реферальных ссылок.
14. Если реферальные ссылки неуместны — стоят универсальные кнопки.

16. Источники чередуются.
17. Жанры чередуются.
18. Темы не повторяются.
19. Медиа релевантно или сгенерировано.

Бот должен быть инструментом роста travel-канала.
