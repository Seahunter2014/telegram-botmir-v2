# Архитектурные решения

1. Проект оставлен в простой структуре `src/`, а не разбит на чрезмерно сложные пакеты.
2. Есть `main.py`, но основной старт Railway — `python -m src.telegram_app`.
3. OpenAI используется при наличии ключа; если API недоступен, включён локальный fallback writer, чтобы бот не молчал.
4. Если фото не найдено через API, бот создаёт локальную travel-картинку через Pillow.
5. Если caption с фото длинный, пост делится на 2 части: первая с фото и фразой «Продолжение следует 👇», вторая с продолжением, CTA и хештегами.
6. Источники обходятся через source manager с round-robin и таймаутами.
7. Fallback-темы хранят историю углов раскрытия, чтобы не повторять смысл.
8. Publisher делает retry и fallback на текст без медиа/кнопок при ошибке Telegram.
